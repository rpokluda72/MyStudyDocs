1. Introduction

RxJS (Reactive Extensions for JavaScript) is a library for reactive programming using Observables, making it easier to compose asynchronous or callback-based code. Think of it as Lodash for events and async data streams.

RxJS provides one core type – Observable – and several supporting types (Observer, Subscription, Subject, Schedulers) along with a large set of operators inspired by Array methods (map, filter, reduce, ...) that allow you to handle asynchronous events as collections.

1.2 The Purity Principle

RxJS makes your code less error-prone by using pure functions. Normally you'd create an impure function that can change shared state. RxJS isolates that state using Observables:

// Impure – uses shared variable
let count = 0;
document.addEventListener('click', () => console.log(`Clicked ${++count} times`));

// Pure with RxJS – state is isolated
import { fromEvent, scan } from 'rxjs';

fromEvent(document, 'click')
  .pipe(scan((count) => count + 1, 0))
  .subscribe((count) => console.log(`Clicked ${count} times`));

1.3 First Example: Event to Observable

import { fromEvent, throttleTime, scan } from 'rxjs';

fromEvent(document, 'click')
  .pipe(
    throttleTime(1000),           // at most one click per second
    scan((count) => count + 1, 0) // accumulate click count
  )
  .subscribe((count) => console.log(`Clicked ${count} times`));

2. Observables

Observables are lazy Push collections of multiple values. They fill the missing spot in the following table:

Single value

Multiple values

Pull (consumer decides when)

Function

Iterator

Push (producer decides when)

Promise

Observable

An Observable is lazy – it does nothing until you subscribe. Every new subscription triggers a fresh, independent execution (cold Observable).

2.1 Creating Observables

Use the Observable constructor for full control:

import { Observable } from 'rxjs';

const observable = new Observable((subscriber) => {
  subscriber.next(1);
  subscriber.next(2);
  subscriber.next(3);
  setTimeout(() => {
    subscriber.next(4);
    subscriber.complete();
  }, 1000);
});

2.2 Subscribing to Observables

Subscribing is like calling a function – each call starts an independent execution. The code inside the Observable only runs when you subscribe.

console.log('before subscribe');
observable.subscribe({
  next(x)     { console.log('got value ' + x); },
  error(err)  { console.error('error: ' + err); },
  complete()  { console.log('done'); },
});
console.log('after subscribe');

// Output:
// before subscribe
// got value 1
// got value 2
// got value 3
// after subscribe
// got value 4   (after ~1 second)
// done

2.3 Executing Observables

The code inside the Observable constructor is the "Observable execution". It can deliver three types of notifications:

Notification

Method

Description

Next

subscriber.next(value)

Sends a value to the observer. Can be called multiple times.

Error

subscriber.error(err)

Sends an error. Terminates the execution – no further notifications.

Complete

subscriber.complete()

Signals completion. No more next() calls after this.

import { Observable } from 'rxjs';

const observable = new Observable((subscriber) => {
  subscriber.next(1);
  subscriber.next(2);
  subscriber.complete();     // signals done
  subscriber.next(3);        // IGNORED – never delivered after complete()
});

observable.subscribe({
  next:     (v) => console.log(v),
  complete: ()  => console.log('Completed'),
});
// 1
// 2
// Completed

2.4 Disposing Observables

subscribe() returns a Subscription. Call unsubscribe() to cancel and release resources:
import { interval } from 'rxjs';

const subscription = interval(400).subscribe((x) => console.log(x));

// Cancel after 1.5 seconds
setTimeout(() => subscription.unsubscribe(), 1500);
// 0
// 1
// 2
// 3  (then stops)

You can also provide custom teardown logic inside the Observable:

const observable = new Observable((subscriber) => {
  const id = setInterval(() => subscriber.next('tick'), 1000);

  // Teardown logic – runs on unsubscribe or complete
  return () => clearInterval(id);
});

3. Observer

An Observer is a consumer of values delivered by an Observable. It is simply an object with three optional callback methods:

import { Observable } from 'rxjs';

const observer = {
  next:     (value)  => console.log('Next:', value),
  error:    (err)    => console.error('Error:', err),
  complete: ()       => console.log('Complete'),
};

observable.subscribe(observer);

3.1 Partial Observers

All three callbacks are optional. You can provide just the ones you need:

// Only handle next values
observable.subscribe((value) => console.log(value));

// Handle next and error
observable.subscribe({
  next:  (value) => console.log(value),
  error: (err)   => console.error(err),
});

// Using shorthand positional arguments (deprecated style)
observable.subscribe(
  (value) => console.log(value),    // next
  (err)   => console.error(err),    // error
  ()      => console.log('done')    // complete
);

3.2 The Observer Interface

Method

Called when

After this…

next(value)

Observable emits a new value

Observable continues normally

error(err)

Observable encounters an error

Observable stops – no further callbacks

complete()

Observable finishes successfully

Observable stops – no further callbacks

If an error occurs and no error handler is provided, it is thrown as an uncaught exception. Always handle errors in long-lived streams.

import { throwError } from 'rxjs';

// Always provide error handler for production code
throwError(() => new Error('Something went wrong'))
  .subscribe({
    next:  (v)   => console.log(v),
    error: (err) => console.error('Caught:', err.message), // Caught: Something went wrong
  });

4. Subscription

A Subscription represents the execution of an Observable. It has one important method: unsubscribe(), which cancels the Observable execution and frees any resources it holds.

import { interval } from 'rxjs';

const observable = interval(1000);
const subscription = observable.subscribe((x) => console.log(x));

// Later, cancel the subscription:
subscription.unsubscribe();

4.1 Composite Subscriptions

Subscriptions can be combined together so that a single unsubscribe() call cancels multiple Observables:

import { interval } from 'rxjs';

const sub1 = interval(400).subscribe((x) => console.log('first:',  x));
const sub2 = interval(300).subscribe((x) => console.log('second:', x));

sub1.add(sub2);          // sub2 becomes a child of sub1

setTimeout(() => {
  sub1.unsubscribe();    // also unsubscribes sub2
}, 1000);

4.2 Managing Subscriptions (Best Practices)

Unmanaged subscriptions cause memory leaks. Always clean up, especially in long-lived components:

import { Subject, interval } from 'rxjs';
import { takeUntil } from 'rxjs/operators';

// Pattern 1: takeUntil with a destroy signal (recommended for Angular)
const destroy$ = new Subject<void>();

interval(1000)
  .pipe(takeUntil(destroy$))
  .subscribe(console.log);

// On component destroy:
destroy$.next();
destroy$.complete();

// Pattern 2: store and unsubscribe manually
const sub = interval(1000).subscribe(console.log);
// ...
sub.unsubscribe();

5. Subject

A Subject is a special type of Observable that allows values to be multicasted to many Observers. While plain Observables are unicast (each subscribed Observer owns an independent execution), Subjects are multicast. A Subject is simultaneously an Observable AND an Observer.

Feature

Observable

Subject

Multicast

No – each subscriber gets own execution

Yes – all subscribers share one execution

Push values

Only from inside constructor

From outside via next(), error(), complete()

Late subscribers

Get full replay (cold)

Miss past values (unless BehaviorSubject etc.)

Use case

Data source / pipeline

Event bus, bridging imperative code

5.1 Basic Subject

import { Subject } from 'rxjs';

const subject = new Subject<number>();

subject.subscribe({ next: (v) => console.log(`observerA: ${v}`) });
subject.subscribe({ next: (v) => console.log(`observerB: ${v}`) });

subject.next(1);
subject.next(2);

// Logs:
// observerA: 1
// observerB: 1
// observerA: 2
// observerB: 2

5.2 Subject as Observer

Because a Subject is also an Observer, you can pass it to subscribe() to multicast any cold Observable:

import { Subject, from } from 'rxjs';

const subject = new Subject<number>();

subject.subscribe({ next: (v) => console.log(`observerA: ${v}`) });
subject.subscribe({ next: (v) => console.log(`observerB: ${v}`) });

const observable = from([1, 2, 3]);
observable.subscribe(subject);   // Subject bridges cold -> multicast

// Logs:
// observerA: 1  observerB: 1  observerA: 2  observerB: 2  ...

5.3 BehaviorSubject

BehaviorSubject stores the current value and immediately emits it to every new subscriber. Perfect for representing "state over time" (e.g. current user, selected tab, form values).

import { BehaviorSubject } from 'rxjs';

const subject = new BehaviorSubject(0);   // initial value = 0

subject.subscribe({ next: (v) => console.log(`observerA: ${v}`) });

subject.next(1);
subject.next(2);

subject.subscribe({ next: (v) => console.log(`observerB: ${v}`) });

subject.next(3);

// Logs:
// observerA: 0
// observerA: 1
// observerA: 2
// observerB: 2   <-- late subscriber gets current value immediately
// observerA: 3
// observerB: 3

// Read current value synchronously:
console.log(subject.getValue());   // 3

5.4 ReplaySubject

ReplaySubject records N past values and replays them to every new subscriber. Optionally, a windowTime (ms) can limit how old recorded values can be.

import { ReplaySubject } from 'rxjs';

const subject = new ReplaySubject(3);   // buffer last 3 values

subject.subscribe({ next: (v) => console.log(`observerA: ${v}`) });

subject.next(1);
subject.next(2);
subject.next(3);
subject.next(4);

subject.subscribe({ next: (v) => console.log(`observerB: ${v}`) });

subject.next(5);

// Logs:
// observerA: 1  observerA: 2  observerA: 3  observerA: 4
// observerB: 2  observerB: 3  observerB: 4  <-- gets last 3 on subscribe
// observerA: 5  observerB: 5

// With window time – replay items from the last 500ms:
const timedSubject = new ReplaySubject(100, 500);

5.5 AsyncSubject

AsyncSubject emits only the last value of the execution, and only when complete() is called. Useful for representing a single async result (similar to Promise).

import { AsyncSubject } from 'rxjs';

const subject = new AsyncSubject<number>();

subject.subscribe({ next: (v) => console.log(`observerA: ${v}`) });

subject.next(1);
subject.next(2);
subject.next(3);
subject.next(4);

subject.subscribe({ next: (v) => console.log(`observerB: ${v}`) });

subject.next(5);
subject.complete();

// Logs:
// observerA: 5
// observerB: 5

6. Scheduler

A Scheduler controls when a subscription starts and when notifications are delivered. It has three components: a data structure (task queue), an execution context (where/when tasks run), and a virtual clock (now()).

6.2 observeOn – Scheduling Notifications

observeOn() controls on which scheduler downstream operators and the subscriber run. The example below shows how asyncScheduler defers delivery to the next event loop tick:
import { Observable, observeOn, asyncScheduler } from 'rxjs';

const observable = new Observable((observer) => {
  observer.next(1);
  observer.next(2);
  observer.next(3);
  observer.complete();
}).pipe(
  observeOn(asyncScheduler)
);

console.log('before subscribe');
observable.subscribe({
  next(x)    { console.log('got value ' + x); },
  complete() { console.log('done'); },
});
console.log('after subscribe');

// Output:
// before subscribe
// after subscribe       <-- note: before the values!
// got value 1
// got value 2
// got value 3
// done

6.3 subscribeOn – Scheduling the Subscription

subscribeOn() schedules the subscribe() call itself on a given scheduler:
import { of, subscribeOn, asyncScheduler } from 'rxjs';

console.log('start');

of(1, 2, 3)
  .pipe(subscribeOn(asyncScheduler))
  .subscribe(console.log);

console.log('end');

// Output:
// start
// end
// 1  2  3  (deferred to next tick)

6.4 Schedulers in Creation Operators

Most creation operators that deal with time accept a scheduler as their last argument:

import { interval, asyncScheduler, asapScheduler } from 'rxjs';

// Default: asyncScheduler
interval(1000).subscribe(console.log);

// Explicit scheduler
interval(1000, asyncScheduler).subscribe(console.log);

// Use asapScheduler for micro-task timing
import { of } from 'rxjs';
of(1, 2, 3, asapScheduler).subscribe(console.log);

7. Operators

Operators are the essential pieces that allow complex async code to be composed in a declarative manner. All pipeable operators are pure functions: they take an Observable and return a new Observable without modifying the source.

import { of } from 'rxjs';
import { map, filter } from 'rxjs/operators';

of(1, 2, 3, 4, 5)
  .pipe(
    filter((n) => n % 2 !== 0),    // 1, 3, 5
    map((n) => n * n)              // 1, 9, 25
  )
  .subscribe((n) => console.log(n));

// map – double each value
of(1, 2, 3).pipe(map((x) => x * 2)).subscribe(console.log);   // 2 4 6

// scan – running total (like reduce but emits each step)
of(1, 2, 3, 4)
  .pipe(scan((acc, val) => acc + val, 0))
  .subscribe(console.log);   // 1 3 6 10

// mergeMap – parallel inner subscriptions (e.g. parallel HTTP)
const ids = of(1, 2, 3);
ids
  .pipe(mergeMap((id) => fetch(`/api/item/${id}`).then((r) => r.json())))
  .subscribe(console.log);

// switchMap – cancel previous (e.g. typeahead search)
fromEvent(searchInput, 'input')
  .pipe(switchMap((e) => fetch(`/api/search?q=${e.target.value}`)))
  .subscribe(console.log);

// concatMap – sequential, preserves order (e.g. save operations)
clicks$
  .pipe(concatMap(() => interval(1000).pipe(take(3))))
  .subscribe(console.log);

Choosing a mapping operator: mergeMap = parallel/concurrent | switchMap = cancel previous | concatMap = sequential | exhaustMap = ignore while busy

7.5 Error Handling Operators

Operator

Description

Intercept error and return a replacement Observable (or rethrow)

import { of, throwError, interval } from 'rxjs';
import { catchError, retry, retryWhen, delayWhen, take } from 'rxjs/operators';

// catchError – recover with a fallback Observable
throwError(() => new Error('oops'))
  .pipe(
    catchError((err) => {
      console.error('Caught:', err.message);
      return of('fallback value');
    })
  )
  .subscribe(console.log);   // fallback value

// retry – simple retry
someApiCall$
  .pipe(retry(3))
  .subscribe(console.log);

// retryWhen – exponential back-off
someApiCall$
  .pipe(
    retryWhen((errors) =>
      errors.pipe(
        delayWhen((_, attempt) => timer(Math.pow(2, attempt) * 1000)),
        take(4)
      )
    )
  )
  .subscribe(console.log);

cachedUser$.subscribe((user) => renderHeader(user));

  .pipe(finalize(() => console.log('Stream ended (any reason)')))
  .subscribe();

7.7 Multicasting Operators

share() and shareReplay() are the modern way to multicast. They internally use a Subject to share one Observable execution among multiple subscribers.
import { interval } from 'rxjs';
import { share, shareReplay, take } from 'rxjs/operators';

// Without share – two subscriptions = two separate interval timers
const cold$ = interval(1000).pipe(take(3));
cold$.subscribe((v) => console.log('sub1:', v));
cold$.subscribe((v) => console.log('sub2:', v));

// With share – both share one timer
const hot$ = interval(1000).pipe(take(3), share());
hot$.subscribe((v) => console.log('sub1:', v));
hot$.subscribe((v) => console.log('sub2:', v));

// shareReplay(1) – replay latest to late subscribers (great for HTTP)
const data$ = from(fetch('/api/data')).pipe(shareReplay(1));
data$.subscribe((d) => renderA(d));
data$.subscribe((d) => renderB(d));   // no second HTTP request

Accumulate all values, emit single result on complete

import { of } from 'rxjs';
import { count, max, min, reduce } from 'rxjs/operators';

const numbers$ = of(5, 4, 7, 2, 8);

numbers$.pipe(count()).subscribe(console.log);                    // 5
numbers$.pipe(max()).subscribe(console.log);                      // 8
numbers$.pipe(min()).subscribe(console.log);                      // 2
numbers$.pipe(reduce((acc, v) => acc + v, 0)).subscribe(console.log); // 26
