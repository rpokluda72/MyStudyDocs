MongoDB Query Language — MQL

Beginner Tutorial · E-commerce / Bookstore Schema

1. Introduction

MongoDB is a document-oriented NoSQL database that stores data as flexible JSON-like documents called BSON. Unlike relational databases there is no fixed schema — documents in the same collection can have different fields.

MQL (MongoDB Query Language) is the interface for querying and manipulating data. Commands are written in JavaScript-like syntax and run in the MongoDB Shell (mongosh) or via a driver (Node.js, Python, Java …).

Key concepts:

  • Document — a single record (like a row in SQL), stored as BSON
  • Collection — a group of documents (like a table in SQL), no fixed schema
  • Database — a named container for collections
  • _id — every document must have a unique _id field; MongoDB auto-generates an ObjectId if you omit it
  • mongosh — the interactive MongoDB shell used for all examples below

3.1 Create — insertOne() and insertMany()

insertOne() adds a single document. insertMany() adds multiple documents in one call. MongoDB auto-generates _id (as ObjectId) when you don't provide it.

insertOne

// Add one product to the products collection
db.products.insertOne({
name: "USB-C Hub",
price: 29.99,
category: "Electronics",
stock: 200,
tags: ["usb", "hub", "accessories"],
description: "7-in-1 USB-C hub with HDMI and card reader"
})
// Returns: { acknowledged: true, insertedId: ObjectId("...") }

insertMany

// Add multiple customers at once
db.customers.insertMany([
{
name: "Bob Smith",
email: "bob@example.com",
address: { city: "Brno", country: "CZ" }
},
{
name: "Carol White",
email: "carol@example.com",
address: { city: "Vienna", country: "AT" }
}
])
// Returns: { insertedCount: 2, insertedIds: { 0: ObjectId(...), 1: ObjectId(...) } }

3.2 Read — findOne() and find()

findOne() returns the first matching document (or null). find() returns a cursor over all matching documents. An empty filter {} matches every document.

findOne / find

// Find the first product with this name
db.products.findOne({ name: "Wireless Keyboard" })

// Find ALL products in the Electronics category
db.products.find({ category: "Electronics" })

// Find everything in the collection (no filter)
db.products.find({})

Projection — choose which fields to return

The second argument to find() is a projection. Use 1 to include a field and 0 to exclude it. _id is included by default unless you explicitly set it to 0.

Projection

// Return only name and price, hide _id
db.products.find(
{ category: "Electronics" },
{ name: 1, price: 1, _id: 0 }
)

// Return everything EXCEPT description and tags
db.products.find(
{},
{ description: 0, tags: 0 }
)

3.3 Update — updateOne() and updateMany()

Updates require two arguments: a filter and an update operator. Never pass a plain document as the second argument — always use an update operator like $set.

Operator

Effect

Example

$set

Set or add a field value

$set: { price: 44.99 }

$unset

Remove a field

$unset: { onSale: '' }

$inc

Increment (or decrement) a number

$inc: { stock: -1 }

$push

Append a value to an array

$push: { tags: 'bluetooth' }

$pull

Remove a value from an array

$pull: { tags: 'peripherals' }

$rename

Rename a field

$rename: { oldName: 'newName' }

updateOne — $set, $inc, $push, $pull

// Change the price of one product
db.products.updateOne(
{ name: "Wireless Keyboard" }, // filter: which document
{ $set: { price: 44.99 } } // update: what to change
)

// Decrease stock by 1 each time a product is sold
db.products.updateOne(
{ name: "Wireless Keyboard" },
{ $inc: { stock: -1 } }
)

// Add a new tag to the tags array
db.products.updateOne(
{ name: "Wireless Keyboard" },
{ $push: { tags: "bluetooth" } }
)

// Remove a tag from the array
db.products.updateOne(
{ name: "Wireless Keyboard" },
{ $pull: { tags: "peripherals" } }
)

updateMany + upsert

// Mark ALL Electronics products as on sale
db.products.updateMany(
{ category: "Electronics" },
{ $set: { onSale: true } }
)

// upsert: update the document if it exists, INSERT it if not
db.products.updateOne(
{ name: "Smart Watch" },
{ $set: { price: 199.99, category: "Electronics", stock: 50 } },
{ upsert: true }
)

3.4 Delete — deleteOne() and deleteMany()

deleteOne() removes the first matching document. deleteMany() removes all matches. Use {} as filter to remove all documents in the collection.

deleteOne / deleteMany

// Remove one specific product
db.products.deleteOne({ name: "Old Product" })

// Remove all out-of-stock products
db.products.deleteMany({ stock: 0 })

// DANGER: remove ALL documents in the collection
db.products.deleteMany({})

4. Querying & Filtering

4.1 Comparison Operators

Operator

Meaning

Example

$eq

Equal (default behaviour)

{ price: { $eq: 49.99 } }

$ne

Not equal

{ category: { $ne: 'Books' } }

$gt

Greater than

{ price: { $gt: 30 } }

$gte

Greater than or equal

{ price: { $gte: 30 } }

$lt

Less than

{ price: { $lt: 100 } }

$lte

Less than or equal

{ price: { $lte: 100 } }

$in

Value is in a list

{ category: { $in: ['Electronics', 'Books'] } }

$nin

Value is NOT in a list

{ category: { $nin: ['Toys', 'Clothing'] } }

Comparison examples

// Products priced above 30
db.products.find({ price: { $gt: 30 } })

// Products priced between 20 and 100 (inclusive)
db.products.find({ price: { $gte: 20, $lte: 100 } })

// Products NOT in the Electronics category
db.products.find({ category: { $ne: "Electronics" } })

// Products in one of these categories
db.products.find({ category: { $in: ["Electronics", "Books"] } })

// Only products that are in stock
db.products.find({ stock: { $gt: 0 } })

4.2 Logical Operators — $and, $or, $nor

$and, $or, $nor

// $and — Electronics AND price under 50
db.products.find({
$and: [
{ category: "Electronics" },
{ price: { $lt: 50 } }
]
})

// Shorthand (implicit $and when combining different fields)
db.products.find({ category: "Electronics", price: { $lt: 50 } })

// $or — Electronics OR Books
db.products.find({
$or: [
{ category: "Electronics" },
{ category: "Books" }
]
})

// $nor — neither out-of-stock NOR discontinued
db.products.find({
$nor: [
{ stock: 0 },
{ discontinued: true }
]
})

4.3 Querying Nested Documents — Dot Notation

Use "field.subfield" syntax (must be in quotes as a string) to query nested fields.

Dot notation — nested documents

// Customers from Prague
db.customers.find({ "address.city": "Prague" })

// Customers from Czech Republic
db.customers.find({ "address.country": "CZ" })

// Combine nested + top-level filter
db.customers.find({ "address.country": "CZ", name: { $ne: "Bob Smith" } })

4.4 Array Queries

When a field holds an array, MongoDB can match inside it without any special unwinding.

Array queries

// Products that have "wireless" anywhere in the tags array
db.products.find({ tags: "wireless" })

// Products with BOTH "wireless" AND "keyboard" in tags
db.products.find({ tags: { $all: ["wireless", "keyboard"] } })

// $elemMatch — at least one array element satisfies ALL conditions
// (needed when you have multiple conditions on the same array element)
db.orders.find({
items: { $elemMatch: { qty: { $gt: 2 }, price: { $lt: 60 } } }
})

// Query by array size — orders with exactly 3 line items
db.orders.find({ items: { $size: 3 } })

5. Sorting, Limiting, Skipping

Chain sort(), limit(), and skip() after find() to control order and page size.

Method

Description

Values

sort({})

Order results by a field

1 = ascending (A→Z, low→high), -1 = descending

limit(n)

Return at most n documents

e.g. limit(10)

skip(n)

Skip the first n documents

e.g. skip(20) = page 3 when limit is 10

sort / limit / skip

// Most expensive products first
db.products.find().sort({ price: -1 })

// Cheapest Electronics first, return max 5 results
db.products.find({ category: "Electronics" }).sort({ price: 1 }).limit(5)

// Sort by two fields: category ascending, then price descending within category
db.products.find().sort({ category: 1, price: -1 })

// Pagination pattern — 10 results per page
// Page 1: skip(0).limit(10)
// Page 2: skip(10).limit(10)
// Page 3: skip(20).limit(10)
db.products.find()
.sort({ name: 1 })
.skip(20)
.limit(10)

6. Full-Text Search

MongoDB supports full-text search via a text index. Create the index once; then use the $text operator in your queries.

Text search

// Step 1: create a text index on the fields you want to search
db.products.createIndex({ name: "text", description: "text" })

// Step 2: search for documents containing "wireless" or "keyboard"
db.products.find({ $text: { $search: "wireless keyboard" } })

// Search for an exact phrase (wrap it in escaped quotes)
db.products.find({ $text: { $search: ""long battery"" } })

// Include "keyboard" but exclude "gaming"
db.products.find({ $text: { $search: "keyboard -gaming" } })

// Combine text search with another filter
db.products.find({
$text: { $search: "wireless" },
price: { $lt: 80 }
})

// Sort by relevance score (best match first)
db.products.find(
{ $text: { $search: "wireless keyboard" } },
{ score: { $meta: "textScore" } } // add score field to results
).sort({ score: { $meta: "textScore" } })

7. Aggregation Pipeline

The aggregation pipeline transforms documents through a series of stages. Each stage receives the output of the previous stage.

Stage

Purpose

SQL equivalent

$match

Filter documents (use early!)

WHERE

$group

Group docs and compute values

GROUP BY + aggregates

$sort

Order results

ORDER BY

$project

Reshape / select fields

SELECT

$limit

Keep only the first N results

LIMIT

$skip

Discard the first N results

OFFSET

$unwind

Expand an array into documents

(flatten a 1-to-many)

$lookup

Join from another collection

LEFT JOIN

$count

Count remaining documents

COUNT(*)

$addFields

Add computed fields

SELECT expr AS alias

Example 1 — Product count and average price per category

$group + $sort

db.products.aggregate([
{
$group: {
_id: "$category", // group by category
count: { $sum: 1 }, // count docs in each group
avgPrice: { $avg: "$price" } // average price
}
},
{ $sort: { count: -1 } } // most products first
])

Example 2 — Total revenue per category (completed orders only)

$match → $unwind → $lookup → $group → $sort → $project

db.orders.aggregate([
// Stage 1: filter — only completed orders
{ $match: { status: "completed" } },

// Stage 2: flatten the items array (one doc per line item)
{ $unwind: "$items" },

// Stage 3: join with products to get the category field
{ $lookup: {
from: "products",
localField: "items.productId",
foreignField: "_id",
as: "product"
}},
{ $unwind: "$product" },

// Stage 4: group by category, compute revenue and count
{ $group: {
_id: "$product.category",
totalRevenue: { $sum: { $multiply: ["$items.price", "$items.qty"] } },
orderCount: { $sum: 1 }
}},

// Stage 5: sort by revenue descending
{ $sort: { totalRevenue: -1 } },

// Stage 6: clean up output — rename _id → category
{ $project: {
_id: 0,
category: "$_id",
totalRevenue: 1,
orderCount: 1
}}
])

Example 3 — Top 3 customers by total spending

Top-N with $lookup

db.orders.aggregate([
{ $match: { status: "completed" } },
{ $group: {
_id: "$customerId",
totalSpent: { $sum: "$total" },
orderCount: { $sum: 1 }
}},
{ $sort: { totalSpent: -1 } },
{ $limit: 3 },
// Join with customers collection to get the name
{ $lookup: {
from: "customers",
localField: "_id",
foreignField: "_id",
as: "customer"
}},
{ $unwind: "$customer" },
{ $project: {
_id: 0,
name: "$customer.name",
totalSpent: 1,
orderCount: 1
}}
])
