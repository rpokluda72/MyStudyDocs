20 nejčastějších otázek a odpovědí o programování v jazyce Java.

  1. Co odlišuje procedurální programování od objektově orientovaného programování?

Procedurální programování je založeno na funkcích se zaměřením na sekvenční provádění úloh. Objektově orientované programování (OOP) strukturuje kód do opakovaně použitelných objektů, které kombinují data (pole) a chování (metody). OOP nabízí pokročilé funkce, jako je dědičnost, polymorfismus a zapouzdření, což umožňuje vývojářům vytvářet škálovatelný a modulární kód.

  1. Co odlišuje abstraktní třídu od rozhraní?

Abstraktní třída může zahrnovat jak abstraktní metody (bez implementace), tak konkrétní metody. Umožňuje stát prostřednictvím polí a podporuje dědičnost. Rozhraní od Java 8 mohou mít výchozí a statické metody, což umožňuje omezenou implementaci. Počínaje verzí Java 9 mohou rozhraní mezi výchozí metody zahrnovat také soukromé metody pro sdílenou logiku.

  1. Jak JVM funguje a co to je?

Java Virtual Machine (JVM) spouští bajtový kód Java na různých platformách a umožňuje funkci „zapsat jednou, spustit kdekoli“. K optimalizaci provádění kódu používá komponenty, jako je zavaděč tříd (načítání tříd za běhu), správa paměti (hromada a zásobník) a kompilátor Just-In-Time (JIT).

  1. Co odlišuje paměť haldy od paměti zásobníku?

Paměť haldy ukládá objekty a jejich přidružená data, spravovaná pomocí garbage collection, aby se znovu získala nevyužitá paměť. Zásobníková paměť se používá pro volání metod a lokálních proměnných podle struktury Last-In-First-Out (LIFO). Zavedení pokročilých garbage collectorů, jako je ZGC v Javě 11, zlepšuje správu haldy.

  1. Co odlišuje zaškrtnuté výjimky od nekontrolovaných?

Kontrolované výjimky (např. IOException) musí být explicitně zpracováno při použití v době kompilace try-catch nebo prohlásil s throws. Nezaškrtnuté výjimky (např. NullPointerException) se vyskytují během běhu a znamenají chyby programování. Osvědčené postupy nyní podporují používání moderních přístupů, jako je např Optional pro lepší zpracování chyb.

  1. Jak vytvoříte singletonovou třídu v Javě? K čemu se používá?

Použijte soukromý konstruktor, soukromou proměnnou statické instance a veřejnou statickou metodu (např getInstance()), abyste instanci vrátili. Počínaje Java 5 je vzor Enum Singleton preferovanou implementací pro svou bezpečnost vláken a jednoduchost. Třídy Singleton se běžně používají pro protokolování, ukládání do mezipaměti nebo správu sdílených zdrojů.

  1. Co odlišuje equals() a == od sebe?

Jedno == operátor porovnává odkazy (zda dvě proměnné ukazují na stejný objekt v paměti). The equals() metoda porovnává logickou rovnost, jako je obsah řetězců nebo vlastních objektů. Přepisování equals() je zásadní při implementaci tříd, jako jsou vlastní datové struktury.

  1. Můžete popsat rozdíl mezi Java HashMap a Hashtable?

HashMap je nesynchronizovaný, umožňuje jeden klíč null a více hodnot null, takže je rychlejší pro aplikace s jedním vláknem. Hashtable je synchronizovaný a bezpečný pro vlákna, ale neumožňuje nulové klíče nebo hodnoty. Pro souběžné scénáře, ConcurrentHashMap (představeno v Javě 5) je moderní náhradou.

  1. Jak se procesy a vlákna od sebe liší?

Proces je nezávislý vykonávající program s vlastním paměťovým prostorem. Vlákno je menší jednotka provádění v rámci procesu, která sdílí paměť s jinými vlákny. Java Virtual Threads (Project Loom v Javě 19) zefektivňuje multithreading tím, že drasticky snižuje režii tradičních vláken.

  1. Můžete vysvětlit klíčové slovo final ?

Jedno final klíčové slovo omezuje změny:

    • final proměnná: Po inicializaci nelze znovu přiřadit.
    • final metoda: Nelze přepsat v podtřídách.
    • final třída: Nelze dědit.
      Zajišťuje neměnnost a bezpečnost, zejména v API.
  1. Co odlišuje statickou vnitřní třídu Java od nestatické vnitřní třídy?

Statická vnitřní třída nezávisí na instanci vnější třídy a může přistupovat pouze ke statickým členům vnější třídy. Nestatická vnitřní třída vyžaduje instanci vnější třídy a má přístup ke všem jejím členům, včetně soukromých.

  1. Co odlišuje ArrayList od LinkedList v Javě?

ArrayList používá dynamické pole a poskytuje přístupový čas O(1) pro náhodné prvky, ale vkládání/mazání (kromě konce) je pomalejší. LinkedList používá dvojitě propojený seznam, takže vkládání a mazání je rychlejší (O(1)), ale pomalejší přístup (O(n)).

  1. Co odlišuje smyčku while v jazyce Java od smyčky do-while?

A while smyčka vyhodnotí podmínku před provedením těla, což znamená, že se nemusí spustit, pokud je podmínka zpočátku nepravdivá. A do-while cyklus se provede alespoň jednou, protože podmínka je kontrolována za tělem.

  1. Čím se liší metody wait() a sleep() v Javě?

Jedno wait() metoda pozastaví vlákno, dokud ho jiné vlákno neupozorní a uvolní zámek na objektu. The sleep() metoda pozastaví vlákno na určenou dobu bez uvolnění zámků.

  1. Můžete popsat rozdíl mezi třídou Java a objektem?

Třída je plán definující strukturu (pole a metody) objektů. Objekt je instancí třídy s pamětí alokovanou pro její konkrétní stav a chování.

  1. Co v Javě odlišuje konstruktor od metody?

Konstruktor inicializuje objekt a sdílí stejný název jako třída. Nemá návratový typ. Metoda definuje chování objektů, má specifický návratový typ a lze ji vyvolat vícekrát.

  1. Můžete objasnit rozdíl mezi statickou a dynamickou vazbou Java?

Ke statické vazbě dochází v době kompilace pro statické, soukromé a konečné metody. Dynamická vazba nastává za běhu a řeší přepsané metody založené na skutečném typu objektu.

  1. Co je polymorfismus a jak jej Java používá?

Polymorfismus umožňuje objektům chovat se odlišně na základě jejich typu. Java podporuje polymorfismus prostřednictvím:

    • Přetížení metody: Polymorfismus v době kompilace, kde metody mají stejný název, ale různé seznamy parametrů.
    • Přepsání metody: Polymorfismus běhového prostředí, kde podtřída poskytuje svou implementaci metody rodičovské třídy.
  1. Můžete objasnit rozdíl mezi hlubokými a mělkými kopiemi Java?

Mělká kopie zkopíruje odkazy na původní objekty, takže změny originálu se projeví v kopii. Hluboká kopie duplikuje objekty a jejich data a vytváří nezávislé instance. Používejte knihovny jako ObjectMapper nebo vlastní logiku pro hluboké klonování.

  1. Můžete popsat rozdíl mezi objektem Java a referencí?

Objekt je instance v paměti haldy obsahující data a metody. Reference je proměnná, která ukládá paměťovou adresu objektu a umožňuje s ním interakci.

Často kladené otázky (FAQ) o Javě

1. K čemu slouží programování v Javě?

Java je velmi flexibilní jazyk, který lze použít pro cokoli od aplikací na straně serveru přes hry až po desktopové a mobilní aplikace. Většina aplikací pro Android, obchodních aplikací a velkých projektů je napsána v Javě.

2. Jaký je rozdíl mezi JDK a JRE?

Java Development Kit (JDK) je sada nástrojů pro vytváření programů založených na Javě. Dodává se s kompilátorem Java, běhovým prostředím Java a dalšími nástroji nezbytnými pro vývoj v Javě. Naopak JRE (Java Runtime Environment) je běhové prostředí potřebné ke spouštění programů Java. Součástí je Java SE Runtime Environment (JRE) a podpůrné knihovny.

3. Jaký je rozdíl mezi rozhraním a abstraktní třídou?

Rozhraní Java a abstraktní třídy slouží stejnému účelu k dosažení abstrakce, ale dělají to různými způsoby. Smlouva nebo sada pokynů, které musí třída dodržovat, je definována jejím rozhraním. Podpisy metody a konstanty jsou všechny přítomné; není zahrnut žádný skutečný kód. Naopak abstraktní třída může zahrnovat jak abstraktní, tak neabstraktní metody, stejně jako implementační kód. Aby bylo možné použít abstraktní třídu, musí ji podtřída rozšířit.

  • 4. Jaký je rozdíl mezi třídou a objektem?

Třídy v Javě jsou jako plány nebo šablony; specifikují, jak by měl být objekt konstruován a jaké metody by měl implementovat. V podstatě obsahuje všechny funkce a operace, které bude mít instance této třídy. Naproti tomu objekt je dynamicky vytvářená instance třídy. Každá instance třídy má svou vlastní jedinečnou hodnotu pro každý atribut a může provádět libovolnou z metod třídy.

  • 5. Co je statická metoda v Javě?

Statická metoda v Javě je metoda, která je spojena se samotnou třídou, spíše než s jakoukoli konkrétní instancí třídy. Bez předchozího vygenerování instance třídy může tato třída pracovat pouze se statickými proměnnými a dalšími statickými metodami. Použijte název třídy následovaný názvem metody k volání statické metody, která byla zadána pomocí klíčového slova static.
