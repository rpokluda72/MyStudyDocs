Building Reusable Components with Input and Output in Angular

Why Reusability Matters?

When working on an Angular application, you often need to share data between components. Instead of duplicating logic, we can create reusable components that handle data dynamically using @Input() and @Output().

Understanding @Input()

The @Input() decorator allows a parent component to pass data to a child component.

Example: Passing a user object to a child component.

Parent Component (app.component.ts)

@Component({ selector: 'app-root', template: `<app-user-card [user]="userData"></app-user-card>` }) export class AppComponent { userData = { name: 'John Doe', age: 30 }; }

Child Component (user-card.component.ts)

@Component({ selector: 'app-user-card', template: `<h3>{{ user.name }}</h3><p>Age: {{ user.age }}</p>` }) export class UserCardComponent { @Input() user!: { name: string; age: number }; }

💡 Now, the UserCard component can be reused anywhere, receiving different user data!

Understanding @Output()

The @Output() decorator allows a child component to emit events to its parent using an EventEmitter.

Example: Child component sending an event when a button is clicked.

Child Component (user-card.component.ts)

@Component({ selector: 'app-user-card', template: ` <h3>{{ user.name }}</h3> <p>Age: {{ user.age }}</p> <button (click)="notifyParent()">Notify Parent</button> ` }) export class UserCardComponent { @Input() user!: { name: string; age: number }; @Output() userClicked = new EventEmitter<string>(); notifyParent() { this.userClicked.emit(this.user.name); } }

Parent Component (app.component.ts)

@Component({ selector: 'app-root', template: ` <app-user-card [user]="userData" (userClicked)="handleUserClick($event)"></app-user-card> ` }) export class AppComponent { userData = { name: 'John Doe', age: 30 }; handleUserClick(userName: string) { console.log(userName + ' was clicked!'); } }

🔹 Now, when the button is clicked, the child component sends data to the parent, making communication seamless!

Best Practices for Reusable Components

✔ Keep them small & focused — One component should serve a single responsibility.
✔ Use strong typing — Define @Input() and @Output() types properly.
✔ Avoid unnecessary @Input() props – Pass only what's needed.
✔ Follow a consistent naming pattern – Use meaningful names like @Output() userClicked instead of @Output() clickEvent.

Conclusion

Using @Input() and @Output(), you can create flexible, maintainable, and reusable components that make your Angular app cleaner and more efficient.
