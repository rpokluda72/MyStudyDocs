1. Introduction

RxJava is the Java implementation of ReactiveX (Reactive Extensions) – a library for composing asynchronous and event-based programs using observable sequences. It extends the classic Observer pattern to support sequences of data/events and adds powerful operators that let you compose, transform, and combine those sequences declaratively.

1.1 The Observable Abstraction

Observables fill a critical gap between synchronous and asynchronous data retrieval:

1.5 Core Principles

Composability: Operators allow you to declaratively chain transformations on async streams, avoiding callback nesting.

Flexibility: Works for single values, finite sequences, and infinite streams – all under one abstraction.

Implementation Agnosticism: RxJava is unopinionated about threading; you choose schedulers without changing business logic.

2. Tutorials – Getting Started

2.1 Dependency (Maven / Gradle)

Add RxJava 3.x to your project:

<!-- Maven -->
<dependency>
    <groupId>io.reactivex.rxjava3</groupId>
    <artifactId>rxjava</artifactId>
    <version>3.1.8</version>
</dependency>

// Gradle
implementation "io.reactivex.rxjava3:rxjava:3.1.8"

2.2 Your First Observable

The simplest way to create an Observable is Observable.just():

import io.reactivex.rxjava3.core.Observable;

Observable<String> observable = Observable.just("Hello", "RxJava", "World");

observable.subscribe(
    item  -> System.out.println("Received: " + item),   // onNext
    error -> System.err.println("Error: " + error),     // onError
    ()    -> System.out.println("Completed!")            // onComplete
);

2.3 Creating with Observable.create()

For full control, use create() and call observer methods manually:

Observable<Integer> source = Observable.create(emitter -> {
    try {
        for (int i = 1; i <= 5; i++) {
            if (emitter.isDisposed()) return;   // check if downstream unsubscribed
            emitter.onNext(i);
        }
        emitter.onComplete();
    } catch (Exception e) {
        emitter.onError(e);
    }
});

source.subscribe(System.out::println);

2.4 Transforming Items with map() and filter()

Observable.range(1, 10)
    .filter(n -> n % 2 == 0)          // keep even numbers
    .map(n -> "Number: " + n)          // transform to String
    .subscribe(System.out::println);

2.5 Threading: subscribeOn & observeOn

subscribeOn() controls on which thread the Observable does its work. observeOn() controls on which thread downstream operators and the subscriber run.
Observable.fromCallable(() -> fetchDataFromNetwork())   // slow I/O
    .subscribeOn(Schedulers.io())          // run fetchData on I/O thread
    .observeOn(Schedulers.single())        // process result on single thread
    .subscribe(
        data  -> updateUI(data),
        error -> showError(error)
    );

// Common Schedulers:
// Schedulers.io()           - for I/O operations (network, file)
// Schedulers.computation()  - for CPU-intensive work
// Schedulers.newThread()    - creates a new thread each time
// Schedulers.single()       - single background thread
// Schedulers.trampoline()   - queues work on current thread

2.6 Disposables and Resource Cleanup

A Disposable lets you cancel a subscription to stop receiving items and free resources:

Disposable disposable = Observable.interval(1, TimeUnit.SECONDS)
    .subscribe(tick -> System.out.println("Tick: " + tick));

// Later, when you no longer need the stream:
disposable.dispose();

// CompositeDisposable - manage multiple subscriptions together
CompositeDisposable composite = new CompositeDisposable();
composite.add(disposable1);
composite.add(disposable2);
composite.dispose();   // disposes all at once

2.7 Flowable and Backpressure

Use Flowable when the producer can emit faster than the consumer can process. BackpressureStrategy controls what happens when the buffer overflows:

Flowable.range(1, 1_000_000)
    .onBackpressureBuffer()         // buffer excess items
    .observeOn(Schedulers.computation())
    .subscribe(System.out::println);

// BackpressureStrategy options:
// BUFFER  - buffer all items (risk OutOfMemoryError)
// DROP    - drop items that cannot be handled
// LATEST  - keep only the latest item
// ERROR   - throw MissingBackpressureException
// MISSING - no strategy; use onBackpressureXxx() downstream

3. Operators

Operators are the building blocks of RxJava pipelines. They transform, filter, combine, and manage Observable sequences. Below are all categories from https://reactivex.io/documentation/operators.html

3.1 Creating Observables

Operators that originate new Observables.

Operator

Description

create()

Create Observable from scratch by calling observer methods programmatically

defer()
Do not create Observable until subscriber subscribes; create fresh one per subscriber
empty()

Create Observable that emits nothing and completes immediately

never()

Create Observable that never emits and never completes

error()

Create Observable that immediately signals an error

from*()

Convert Iterable, array, Future, Callable, or another reactive type into Observable

interval()

Emit a sequence of integers spaced by a given time interval

just()

Emit one to nine specific items (constant values)

range()

Emit a range of sequential integers

repeat()

Emit a particular item or sequence of items repeatedly

timer()

Emit a single item after a given delay

generate()

Create a cold Observable with a stateful generator function

// just - emit pre-existing values
Observable.just(1, 2, 3).subscribe(System.out::println);

// fromIterable - wrap a collection
Observable.fromIterable(Arrays.asList("a", "b", "c")).subscribe(System.out::println);

// interval - tick every second
Observable.interval(1, TimeUnit.SECONDS).take(5).subscribe(System.out::println);

// defer - fresh Observable per subscription
Observable<Long> deferred = Observable.defer(() -> Observable.just(System.currentTimeMillis()));

3.2 Transforming Observables

Operators that transform the items emitted by an Observable.

Operator

Description

map()

Transform each item by applying a function

flatMap()

Transform each item into an Observable, then flatten/merge all resulting Observables

concatMap()

Like flatMap but preserves order (no interleaving)

switchMap()

Like flatMap but switches to the latest inner Observable, cancelling previous ones

buffer()

Collect items into bundles (lists) and emit the bundles

window()

Subdivide items into Observable windows instead of lists

groupBy()

Divide Observable into a set of Observables organized by a key

scan()

Apply a function to each item sequentially, emitting each intermediate result

cast()

Cast each item to a different type

// map - transform each item
Observable.just(1, 2, 3)
    .map(n -> n * n)
    .subscribe(System.out::println);  // 1, 4, 9

// flatMap - one item to many items (order not guaranteed)
Observable.just("Hello World", "RxJava")
    .flatMap(sentence -> Observable.fromArray(sentence.split(" ")))
    .subscribe(System.out::println);

// scan - running total
Observable.just(1, 2, 3, 4, 5)
    .scan(0, Integer::sum)
    .subscribe(System.out::println);  // 0, 1, 3, 6, 10, 15

// buffer - emit lists of 3
Observable.range(1, 9)
    .buffer(3)
    .subscribe(System.out::println);  // [1,2,3], [4,5,6], [7,8,9]

3.3 Filtering Observables

Operators that selectively emit items from an Observable.

Operator

Description

filter()

Emit only items that pass a predicate test

first() / firstOrError()

Emit only the first item (or error if empty)

last() / lastOrError()

Emit only the last item

take(n)

Emit only the first n items

takeLast(n)

Emit only the last n items

skip(n)

Skip the first n items, emit the rest

skipLast(n)

Skip the last n items

distinct()

Suppress duplicate items

distinctUntilChanged()

Suppress consecutively duplicate items

elementAt(n)

Emit only the item at position n

ignoreElements()

Emit no items but mirror complete/error notifications

sample()

Emit the most recent item within periodic time intervals

throttleFirst()

Emit the first item in each time window

debounce()

Emit an item only if a specified timespan passes without another emission

timeout()

Signal an error if no item is emitted within a timeout period

// filter - keep only even numbers
Observable.range(1, 10)
    .filter(n -> n % 2 == 0)
    .subscribe(System.out::println);  // 2, 4, 6, 8, 10

// take + skip
Observable.range(1, 10).take(3).subscribe(System.out::println);  // 1, 2, 3
Observable.range(1, 10).skip(7).subscribe(System.out::println);  // 8, 9, 10

// debounce - wait 300ms of silence (search box use case)
searchInputObservable
    .debounce(300, TimeUnit.MILLISECONDS)
    .subscribe(query -> performSearch(query));

// distinct - no duplicates
Observable.just(1, 1, 2, 2, 3, 1)
    .distinct()
    .subscribe(System.out::println);  // 1, 2, 3

3.4 Combining Observables

Operators that work with multiple Observables to create a single Observable.

Operator

Description

merge()

Combine multiple Observables into one by merging their emissions (concurrent)

concat()

Emit items from multiple Observables sequentially (no interleaving)

zip()

Combine emissions from multiple Observables pairwise via a combiner function

combineLatest()

When any source emits, combine with the latest from all other sources

startWith()

Prepend specified items before the source Observable emits

switchOnNext()

Subscribe to the most recently emitted inner Observable, cancelling the previous

join()

Combine items from two Observables based on overlapping time windows

withLatestFrom()

Combine each source item with the latest item from another Observable

amb()

Emit from only the first Observable to emit an item (ambiguous race)

// merge - interleave two streams
Observable<String> o1 = Observable.just("A", "B");
Observable<String> o2 = Observable.just("1", "2");
Observable.merge(o1, o2).subscribe(System.out::println);

// zip - pair items from two streams
Observable.zip(
    Observable.just("Alice", "Bob", "Charlie"),
    Observable.just(30, 25, 35),
    (name, age) -> name + " is " + age
).subscribe(System.out::println);
// Alice is 30, Bob is 25, Charlie is 35

// combineLatest - react to either source changing
Observable.combineLatest(usernameObs, passwordObs,
    (u, p) -> !u.isEmpty() && p.length() >= 8)
    .subscribe(valid -> loginButton.setEnabled(valid));

3.5 Error Handling Operators

Operator

Description

onErrorReturn()

Return a default item when an error occurs, then complete

onErrorResumeNext()

Switch to a different Observable when an error occurs

onErrorComplete()

Complete (without error) when an error occurs

retry()

Resubscribe to the source Observable on error, optionally with a count

retryWhen()

Retry with custom logic (e.g., exponential back-off)

doOnError()

Register a side-effect action when an error occurs (does not handle it)

// onErrorReturn - fallback value
Observable.just("data")
    .map(s -> { throw new RuntimeException("oops"); })
    .onErrorReturn(e -> "fallback")
    .subscribe(System.out::println);  // fallback

// retry - resubscribe up to 3 times
Observable.fromCallable(() -> unstableNetworkCall())
    .subscribeOn(Schedulers.io())
    .retry(3)
    .subscribe(result -> process(result), error -> log(error));

// retryWhen - exponential back-off
source.retryWhen(errors -> errors
    .zipWith(Observable.range(1, 3), (e, i) -> i)
    .flatMap(i -> Observable.timer(i, TimeUnit.SECONDS)));

// delay - emit 2 seconds later
Observable.just("delayed")

    .subscribe(System.out::println);

// doOnNext - log side-effects without transforming
Observable.range(1, 5)

    .map(n -> n * 2)

// using - resource tied to Observable lifetime

    conn -> Observable.fromIterable(conn.query("SELECT ...")),

);

3.7 Conditional and Boolean Operators

Operator

Description

all()

Determine if all items meet a criterion (returns Single<Boolean>)

Determine if any item meets a criterion (returns Single<Boolean>)

contains()

Determine if an Observable emits a particular item

Determine if an Observable emits any items

Emit from only the first Observable to emit an item

Emit a default item if the source emits nothing before completing

Determine if two Observables emit the same sequence of items

Observable.just(2, 4, 6).all(n -> n % 2 == 0)

Observable.<String>empty()
    .defaultIfEmpty("No items found")
    .subscribe(System.out::println);

// takeUntil - stop when trigger fires

    .takeUntil(stopSignalObservable)
    .subscribe(System.out::println);

3.8 Mathematical and Aggregate Operators

These operators compute aggregate values over an entire Observable sequence (usually via RxJava extras or built-in methods on Flowable/Observable).

Operator

Description

count()

Count the number of items emitted (returns Single<Long>)

reduce()

Apply a function to each item and emit the final accumulated value

collect()

Collect items into a mutable data structure (returns Single)

toList()

Collect all items into a Single<List<T>>

toMap()

Collect items into a Single<Map<K,V>>

concat()

Emit all items from multiple Observables sequentially without interleaving

sum / average / min / max

Available via rxjava-math library or manual reduce()

// count
Observable.range(1, 100).count()
    .subscribe(c -> System.out.println("Count: " + c));  // 100

// reduce - sum all items
Observable.range(1, 5)
    .reduce(0, Integer::sum)
    .subscribe(sum -> System.out.println("Sum: " + sum));  // 15

// toList - collect into a list
Observable.just("a", "b", "c")
    .toList()
    .subscribe(list -> System.out.println(list));  // [a, b, c]

// concat - sequential, no interleaving
Observable.concat(
    Observable.just(1, 2, 3),
    Observable.just(4, 5, 6)
).subscribe(System.out::println);  // 1, 2, 3, 4, 5, 6

3.9 Connectable Observable Operators

Connectable Observables do not begin emitting upon subscription; they wait for a connect() call. This lets you set up multiple subscribers before any items are emitted.

Operator

Description

publish()

Convert an ordinary Observable into a ConnectableObservable

connect()

Instruct a ConnectableObservable to begin emitting items to subscribers

refCount()
Make a ConnectableObservable behave like an ordinary Observable; auto-connects when first subscriber arrives
replay()

Ensure all observers see the same emitted sequence even if they subscribe late

share()

Shorthand for publish().refCount() - multicast to multiple subscribers with auto-connect

// publish + connect - coordinate multiple subscribers
ConnectableObservable<Long> hot = Observable.interval(1, TimeUnit.SECONDS).publish();
hot.subscribe(t -> System.out.println("Sub1: " + t));
hot.subscribe(t -> System.out.println("Sub2: " + t));
hot.connect();   // both subscribers receive the same ticks

// share - auto-connect multicast
Observable<Long> shared = Observable.interval(1, TimeUnit.SECONDS).share();
shared.subscribe(t -> System.out.println("A: " + t));
shared.subscribe(t -> System.out.println("B: " + t));

// replay - cache and replay to late subscribers
ConnectableObservable<Integer> replayed = Observable.range(1, 5).replay();
replayed.connect();
replayed.subscribe(System.out::println);  // all 5 items replayed to late subscriber

3.10 Backpressure (Flowable)

When a producer emits faster than the consumer can process, use Flowable with a BackpressureStrategy:

Strategy / Operator

Description

BackpressureStrategy.BUFFER

Buffer all outstanding items in memory (risk OutOfMemoryError on unbounded streams)

BackpressureStrategy.DROP

Drop items that the consumer cannot handle

BackpressureStrategy.LATEST

Keep only the most recently emitted item when overwhelmed

BackpressureStrategy.ERROR

Throw MissingBackpressureException when overwhelmed

onBackpressureBuffer()

Apply buffer strategy downstream

onBackpressureDrop()

Apply drop strategy downstream

onBackpressureLatest()

Apply latest strategy downstream

4. Practical Examples

Example 1: Basic Observable Chain

Create, transform, filter, and subscribe.

Observable.range(1, 20)
    .filter(n -> n % 3 == 0)          // multiples of 3
    .map(n -> "Value: " + n)
    .take(4)                           // stop after 4
    .subscribe(
        System.out::println,
        Throwable::printStackTrace,
        () -> System.out.println("Done!")
    );
// Output: Value: 3, Value: 6, Value: 9, Value: 12, Done!

Example 2: Async Network Call

Use fromCallable + subscribeOn to push blocking I/O off the main thread.

public Observable<User> fetchUser(int userId) {
    return Observable.fromCallable(() -> {
        // blocking HTTP call
        return httpClient.get("/users/" + userId, User.class);
    })
    .subscribeOn(Schedulers.io())
    .observeOn(Schedulers.single());
}

// Usage:
fetchUser(42)
    .subscribe(
        user  -> System.out.println("Got user: " + user.getName()),
        error -> System.err.println("Failed: " + error.getMessage())
    );

Example 3: Combining Two Streams with zip()

Pair items from two independent Observables to produce combined results.

Observable<String> names   = Observable.just("Alice", "Bob", "Carol");
Observable<Integer> scores = Observable.just(95, 87, 92);

Observable.zip(names, scores, (name, score) -> name + ": " + score)
    .subscribe(System.out::println);
// Alice: 95
// Bob: 87
// Carol: 92

Example 4: Error Handling with retry and Fallback

Observable.fromCallable(() -> unstableService.getData())
    .subscribeOn(Schedulers.io())
    .retry(3)                                    // retry up to 3 times
    .onErrorReturn(e -> Collections.emptyList()) // if still failing, return empty list
    .subscribe(
        data  -> processData(data),
        error -> log.error("Unrecoverable error", error)
    );

// Exponential back-off retry:
Observable.fromCallable(() -> unstableService.getData())
    .retryWhen(errors -> errors
        .zipWith(Observable.range(1, 4), (err, attempt) -> attempt)
        .flatMap(attempt -> Observable.timer(attempt, TimeUnit.SECONDS))
    )
    .subscribe(System.out::println);

Example 5: Debounce for Search Input

Avoid firing a search on every keystroke – wait for the user to pause typing.

// searchTextChanges is an Observable<String> from a text-input field
searchTextChanges
    .debounce(300, TimeUnit.MILLISECONDS)   // wait for 300ms pause
    .distinctUntilChanged()                  // ignore if same text as before
    .filter(query -> query.length() >= 2)    // skip very short queries
    .switchMap(query ->                      // cancel previous, run new search
        searchService.search(query)
            .subscribeOn(Schedulers.io())
            .onErrorReturnItem(Collections.emptyList())
    )
    .observeOn(Schedulers.single())
    .subscribe(results -> displayResults(results));

Example 6: Subjects – Bridging Imperative and Reactive Code

A Subject acts as both Observer and Observable. Use it to manually push items into a stream from non-reactive code.

// PublishSubject - subscribers only receive items emitted after they subscribe
PublishSubject<String> subject = PublishSubject.create();
subject.subscribe(s -> System.out.println("Sub1: " + s));
subject.onNext("Hello");    // Sub1: Hello
subject.subscribe(s -> System.out.println("Sub2: " + s));
subject.onNext("World");    // Sub1: World, Sub2: World
subject.onComplete();

// BehaviorSubject - replays the last item to new subscribers
BehaviorSubject<Integer> behavior = BehaviorSubject.createDefault(0);
behavior.onNext(1);
behavior.onNext(2);
behavior.subscribe(n -> System.out.println("Late sub: " + n));  // Late sub: 2
behavior.onNext(3);  // Late sub: 3

// ReplaySubject - caches all (or N) items for new subscribers
ReplaySubject<String> replay = ReplaySubject.create();
replay.onNext("A");
replay.onNext("B");
replay.subscribe(System.out::println);  // A, B
replay.onNext("C");                     // C

Example 7: Parallel Execution with flatMap and Schedulers

Process a list of items concurrently on the I/O thread pool.

List<String> urls = Arrays.asList(url1, url2, url3, url4);

Observable.fromIterable(urls)
    .flatMap(url ->
        Observable.fromCallable(() -> downloadFile(url))
            .subscribeOn(Schedulers.io())  // each download on its own I/O thread
    )
    .observeOn(Schedulers.single())
    .toList()
    .subscribe(
        files -> System.out.println("Downloaded " + files.size() + " files"),
        Throwable::printStackTrace
    );
