Java 8

*(March 2014 — the most transformative Java release in years, still the most widely deployed LTS baseline)*

1. Lambda Expressions

Before Java 8, passing behaviour required anonymous inner classes. Lambdas replace them with a concise (params) -> body syntax.

Before Java 8:

List<String> names = Arrays.asList("Charlie", "Alice", "Bob");
Collections.sort(names, new Comparator<String>() {
    @Override
    public int compare(String a, String b) {
        return a.compareTo(b);
    }
});

Java 8+:

List<String> names = Arrays.asList("Charlie", "Alice", "Bob");
Collections.sort(names, (a, b) -> a.compareTo(b));
// or even shorter:
names.sort(String::compareTo);
System.out.println(names); // [Alice, Bob, Charlie]

Lambda shapes:

Runnable r      = () -> System.out.println("no args");
Consumer<String> c = s -> System.out.println(s);       // one arg, no braces
BiFunction<Integer, Integer, Integer> add = (a, b) -> a + b;
Comparator<String> cmp = (a, b) -> {                   // block body
    System.out.println("comparing");
    return a.compareTo(b);
};

A functional interface has exactly one abstract method and can be used as a lambda target. Java 8 ships many in java.util.function:

Use

`T → boolean`

filter

`T → void`

Predicate<String>  nonEmpty = s -> !s.isEmpty();

System.out.println(nonEmpty.test("hi"));   // true

print.accept("world");                     // world
// Compose predicates
Predicate<String> shortAndNonEmpty = nonEmpty.and(s -> s.length() < 5);
System.out.println(shortAndNonEmpty.test("hi")); // true
System.out.println(shortAndNonEmpty.test(""));   // false

Method reference types:

// Instance method on arbitrary instance

// Instance method on specific instance
String prefix = "Hello, ";
Function<String, String> greet = prefix::concat;
// Constructor
Supplier<ArrayList<String>> newList = ArrayList::new;

3. Stream API

Declarative, pipeline-based processing of collections. Streams are lazy — intermediate operations only run when a terminal operation is called.

Before Java 8:

List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "Dave");
List<String> result = new ArrayList<>();
for (String name : names) {
    if (name.length() > 3) {
        result.add(name.toUpperCase());
    }
}
Collections.sort(result);

Java 8+:

import java.util.stream.*;
List<String> names = List.of("Alice", "Bob", "Charlie", "Dave");
List<String> result = names.stream()
    .filter(n -> n.length() > 3)
    .map(String::toUpperCase)
    .sorted()
    .collect(Collectors.toList());
System.out.println(result); // [ALICE, CHARLIE, DAVE]

Common terminal operations:

List<Integer> nums = List.of(1, 2, 3, 4, 5);
int sum   = nums.stream().mapToInt(Integer::intValue).sum();
long cnt  = nums.stream().filter(n -> n % 2 == 0).count();
Optional<Integer> max = nums.stream().max(Integer::compareTo);
boolean anyOver3 = nums.stream().anyMatch(n -> n > 3);
System.out.println(sum);       // 15
System.out.println(cnt);       // 2
System.out.println(max.get()); // 5
System.out.println(anyOver3);  // true

Collectors:

import java.util.stream.Collectors;
List<String> words = List.of("apple", "banana", "avocado", "blueberry");
// group by first letter
Map<Character, List<String>> grouped = words.stream()
    .collect(Collectors.groupingBy(w -> w.charAt(0)));
System.out.println(grouped);
// {a=[apple, avocado], b=[banana, blueberry]}
// join with separator
String joined = words.stream().collect(Collectors.joining(", "));
System.out.println(joined); // apple, banana, avocado, blueberry
// partition by predicate
Map<Boolean, List<String>> partitioned = words.stream()
    .collect(Collectors.partitioningBy(w -> w.length() > 5));
System.out.println(partitioned);
// {false=[apple], true=[banana, avocado, blueberry]}

`flatMap` — flatten nested collections:

List<List<Integer>> nested = List.of(List.of(1, 2), List.of(3, 4), List.of(5));
List<Integer> flat = nested.stream()
    .flatMap(Collection::stream)
    .collect(Collectors.toList());
System.out.println(flat); // [1, 2, 3, 4, 5]

4. Optional

A container that may or may not hold a value. Replaces null returns and forces callers to handle the empty case.

Before Java 8:

public String findUserName(int id) {
    User user = db.find(id);
    return user != null ? user.getName() : null; // caller must remember to null-check
}

Java 8+:

import java.util.Optional;
public Optional<String> findUserName(int id) {
    return Optional.ofNullable(db.find(id)).map(User::getName);
}
// At the call site
findUserName(42)
    .ifPresent(name -> System.out.println("Found: " + name));
String name = findUserName(99).orElse("Unknown");
System.out.println(name); // Unknown
String upper = findUserName(42).map(String::toUpperCase).orElse("NONE");
Optional<String> opt = Optional.of("hello");
System.out.println(opt.isPresent());   // true
System.out.println(opt.get());         // hello
Optional<String> empty = Optional.empty();
System.out.println(empty.isPresent()); // false
System.out.println(empty.orElse("default")); // default

Rule: Never use Optional as a field or method parameter — only as a return type.

5. Default and Static Interface Methods

Interfaces can now have method implementations, allowing APIs to evolve without breaking all implementations.

Before Java 8:

// Adding a method to an interface broke every implementing class
interface Collection<E> {
    void add(E e);
    // can't add forEach() — would break millions of implementations
}

Java 8+:

interface Greeter {
    String greet(String name);                          // abstract
    default String greetLoud(String name) {             // default — can be overridden
        return greet(name).toUpperCase();
    }
    static Greeter formal() {                           // static factory
        return name -> "Good day, " + name;
    }
}
Greeter g = name -> "Hi, " + name;
System.out.println(g.greet("Alice"));      // Hi, Alice
System.out.println(g.greetLoud("Alice"));  // HI, ALICE
System.out.println(Greeter.formal().greet("Bob")); // Good day, Bob

This is how Collection.forEach(), List.sort(), and Map.getOrDefault() were added without breaking existing code.

6. Date/Time API (`java.time`)

Replaces the broken java.util.Date and Calendar with an immutable, thread-safe, and intuitive API inspired by Joda-Time.

Before Java 8 — `Date` and `Calendar` problems:

// Mutable, month is 0-based, not thread-safe
Calendar cal = Calendar.getInstance();
cal.set(2024, 0, 15); // January = 0 (!)
Date date = cal.getTime();

Java 8+:

import java.time.*;
import java.time.format.DateTimeFormatter;
LocalDate date = LocalDate.of(2024, 1, 15);  // month is 1-based
System.out.println(date);                      // 2024-01-15
System.out.println(date.getYear());            // 2024
System.out.println(date.getMonth());           // JANUARY
System.out.println(date.plusDays(10));         // 2024-01-25
LocalTime time = LocalTime.of(14, 30);
System.out.println(time);                      // 14:30
LocalDateTime dt = LocalDateTime.of(date, time);
System.out.println(dt);                        // 2024-01-15T14:30
// With timezone
ZonedDateTime zdt = ZonedDateTime.now(ZoneId.of("Europe/Paris"));
System.out.println(zdt); // 2024-01-15T14:30:00+01:00[Europe/Paris]
// Formatting
DateTimeFormatter fmt = DateTimeFormatter.ofPattern("dd/MM/yyyy");
System.out.println(date.format(fmt));          // 15/01/2024
LocalDate parsed = LocalDate.parse("15/01/2024", fmt);
System.out.println(parsed);                    // 2024-01-15

Durations and Periods:

LocalDate start = LocalDate.of(2024, 1, 1);
LocalDate end   = LocalDate.of(2024, 6, 15);
Period period = Period.between(start, end);
System.out.println(period.getMonths() + " months"); // 5 months
Duration d = Duration.ofHours(2).plusMinutes(30);
System.out.println(d.toMinutes());  // 150

7. CompletableFuture

Async programming with a fluent, composable API. Replaces cumbersome Future.get() blocking calls.

Before Java 8 — blocking `Future`:

ExecutorService ex = Executors.newFixedThreadPool(4);
Future<String> future = ex.submit(() -> fetchData());
String result = future.get(); // BLOCKS the thread until done

Java 8+ — non-blocking chain:

import java.util.concurrent.CompletableFuture;
CompletableFuture<String> cf = CompletableFuture
    .supplyAsync(() -> fetchUser(1))          // run async
    .thenApply(user -> user.toUpperCase())    // transform result
    .thenApply(upper -> "Hello, " + upper);  // chain
cf.thenAccept(System.out::println);  // Hello, ALICE (printed async)
// Combine two independent async tasks
CompletableFuture<String> user  = CompletableFuture.supplyAsync(() -> fetchUser(1));
CompletableFuture<String> order = CompletableFuture.supplyAsync(() -> fetchOrder(1));
CompletableFuture<String> combined = user.thenCombine(order,
    (u, o) -> u + " ordered " + o);
System.out.println(combined.join()); // Alice ordered Pizza

Error handling:

CompletableFuture.supplyAsync(() -> riskyOp())
    .exceptionally(ex -> "fallback value")
    .thenAccept(System.out::println); // fallback value (if exception thrown)

Java 9

1. Module System (JPMS — Project Jigsaw)

The Java Platform Module System lets you declare which packages a module exports and which modules it requires. Improves encapsulation and reduces classpath hell.

// module-info.java (at root of source tree)
module com.example.myapp {
    requires java.net.http;          // depend on another module
    exports com.example.myapp.api;   // only this package is visible to others
    // com.example.myapp.internal is NOT accessible from outside
}

Most developers encounter modules when a library stops working after upgrading — usually because it accessed JDK internals that are now encapsulated. Check if --add-opens flags are needed as a temporary workaround.

2. JShell (REPL)

An interactive Java shell for quick experiments — no need to write a full class.

$ jshell
jshell> int x = 10
x ==> 10
jshell> x * x
$2 ==> 100
jshell> "hello".toUpperCase()
$3 ==> "HELLO"
jshell> /exit

3. Collection Factory Methods

Concise, unmodifiable collection literals.

Before Java 9:

List<String> list = new ArrayList<>(Arrays.asList("a", "b", "c"));
Set<Integer> set  = new HashSet<>(Arrays.asList(1, 2, 3));
Map<String, Integer> map = new HashMap<>();
map.put("x", 1); map.put("y", 2);

Java 9+:

List<String>         list = List.of("a", "b", "c");
Set<Integer>         set  = Set.of(1, 2, 3);
Map<String, Integer> map  = Map.of("x", 1, "y", 2);
System.out.println(list); // [a, b, c]
System.out.println(map);  // {x=1, y=2}
// All are unmodifiable — list.add("d") throws UnsupportedOperationException

For maps with more than 10 entries, use Map.ofEntries():

Map<String, Integer> big = Map.ofEntries(
    Map.entry("a", 1),
    Map.entry("b", 2)
    // ... up to any size
);

4. Stream Improvements

`takeWhile` / `dropWhile` — ordered short-circuiting:

List<Integer> nums = List.of(1, 2, 3, 4, 5, 1);
List<Integer> taken = nums.stream().takeWhile(n -> n < 4).toList();
System.out.println(taken); // [1, 2, 3]  — stops at first element >= 4
List<Integer> dropped = nums.stream().dropWhile(n -> n < 4).toList();
System.out.println(dropped); // [4, 5, 1]

`Stream.iterate` with predicate — like a for-loop:

// Before Java 9 — had to use limit() or filter()
Stream.iterate(0, n -> n + 2).limit(5).forEach(System.out::println);
// 0 2 4 6 8
// Java 9+ — cleaner with a stop condition
Stream.iterate(0, n -> n < 10, n -> n + 2).forEach(System.out::println);
// Output: 0 2 4 6 8

`Stream.ofNullable` — avoids NPE when wrapping a possibly-null value:

String value = null;
Stream.ofNullable(value).forEach(System.out::println); // prints nothing, no NPE

5. Optional Improvements

Optional<String> opt = Optional.empty();
// ifPresentOrElse — handle both cases in one call
opt.ifPresentOrElse(
    v -> System.out.println("found: " + v),
    () -> System.out.println("not found")
);
// Output: not found
// or() — provide an alternative Optional (unlike orElse which provides a value)
Optional<String> result = opt.or(() -> Optional.of("default"));
System.out.println(result.get()); // default
// stream() — treat Optional as a 0- or 1-element stream (useful in flatMap)
long count = opt.stream().count();
System.out.println(count); // 0

6. Private Interface Methods

Interfaces can now have private methods to share code between default methods without exposing it.

interface Validator {
    boolean validate(String s);
    default boolean validateAndLog(String s) {
        boolean result = validate(s);
        log(s, result);        // call private helper
        return result;
    }
    private void log(String s, boolean result) {
        System.out.println("Validated '" + s + "': " + result);
    }
}

Java 10

1. Local Variable Type Inference (`var`)

The compiler infers the type from the right-hand side. Reduces verbosity without losing static typing.

Before Java 10:

HashMap<String, List<Integer>> map = new HashMap<String, List<Integer>>();
BufferedReader reader = new BufferedReader(new FileReader("file.txt"));

Java 10+:

var map    = new HashMap<String, List<Integer>>(); // type inferred
var reader = new BufferedReader(new FileReader("file.txt"));
// Works in loops too
var names = List.of("Alice", "Bob", "Charlie");
for (var name : names) {
    System.out.println(name.toUpperCase()); // Alice, Bob, Charlie
}

Rules: var only works for local variables with an initializer. Cannot be used for fields, method parameters, or return types.

2. Unmodifiable Collection Copies

List<String> mutable = new ArrayList<>(List.of("a", "b", "c"));
mutable.add("d");
// Before Java 10
List<String> copy = Collections.unmodifiableList(new ArrayList<>(mutable));
// Java 10+
List<String> copy = List.copyOf(mutable);   // unmodifiable snapshot
Set<String>  set  = Set.copyOf(mutable);
Map<String, Integer> mapCopy = Map.copyOf(Map.of("x", 1));
System.out.println(copy); // [a, b, c, d]
copy.add("e");            // UnsupportedOperationException

Java 11

*(Long-Term Support — widely used production baseline)*

1. New String Methods

String s = "  Hello World  ";
System.out.println(s.strip());        // "Hello World"  (Unicode-aware, unlike trim())
System.out.println(s.stripLeading()); // "Hello World  "
System.out.println(s.stripTrailing()); // "  Hello World"
System.out.println("".isBlank());     // true
System.out.println("  ".isBlank());   // true
System.out.println("ha".repeat(3));   // hahaha
"line1\nline2\nline3".lines()
    .forEach(System.out::println);
// line1
// line2
// line3

2. HTTP Client (Standard)

A modern replacement for HttpURLConnection, supporting HTTP/1.1 and HTTP/2, sync and async.

Before Java 11:

// HttpURLConnection — verbose and limited
URL url = new URL("https://example.com/api");
HttpURLConnection conn = (HttpURLConnection) url.openConnection();
conn.setRequestMethod("GET");
BufferedReader in = new BufferedReader(new InputStreamReader(conn.getInputStream()));
String line; StringBuilder sb = new StringBuilder();
while ((line = in.readLine()) != null) sb.append(line);
in.close();
String body = sb.toString();

Java 11+:

import java.net.http.*;
import java.net.URI;
var client = HttpClient.newHttpClient();
var request = HttpRequest.newBuilder()
    .uri(URI.create("https://example.com/api"))
    .GET()
    .build();
// Synchronous
HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
System.out.println(response.statusCode()); // 200
System.out.println(response.body());       // response body
// Asynchronous
client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
    .thenApply(HttpResponse::body)
    .thenAccept(System.out::println);

3. `var` in Lambda Parameters

Allows annotations on lambda parameters (the main practical use case).

// Java 11 — needed for annotating lambda params
var list = List.of("a", "bb", "ccc");
list.stream()
    .filter((@NonNull var s) -> !s.isEmpty())
    .forEach(System.out::println);

import java.nio.file.*;
// Write a string to a file
Path file = Path.of("hello.txt");
Files.writeString(file, "Hello, Java 11!");
// Read it back
String content = Files.readString(file);

No need to compile separately for single-file programs:

# Before Java 11
javac Hello.java && java Hello

Java 12

1. Switch Expressions *(Preview)*

switch can now return a value and uses -> arrow syntax — no more fall-through bugs.

Before Java 12 — statement switch with fall-through risk:

int day = 3;
String name;
switch (day) {
    case 1: name = "Monday"; break;
    case 2: name = "Tuesday"; break;
    case 3: name = "Wednesday"; break;
    default: name = "Other"; break;
}

Java 12+ (preview, final in 14):

int day = 3;
String name = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3 -> "Wednesday";
    default -> "Other";
};
System.out.println(name); // Wednesday
// Multiple labels per case
String type = switch (day) {
    case 1, 7 -> "Weekend";
    default   -> "Weekday";
};

2. Teeing Collector

Pipes a stream into two collectors simultaneously, then merges their results.

import java.util.stream.Collectors;
var stats = Stream.of(1, 2, 3, 4, 5)
    .collect(Collectors.teeing(
        Collectors.summingInt(Integer::intValue),   // collector 1: sum
        Collectors.counting(),                       // collector 2: count
        (sum, count) -> "sum=" + sum + " avg=" + (sum / count)
    ));
System.out.println(stats); // sum=15 avg=3

Java 13

1. Text Blocks *(Preview)*

Multi-line strings without escape sequences — finalized in Java 15.

Before:

String html = "<html>\n" +
              "  <body>Hello</body>\n" +
              "</html>";

Java 13+ (preview):

String html = """
        <html>
          <body>Hello</body>
        </html>
        """;
System.out.println(html);
// <html>
//   <body>Hello</body>
// </html>

The closing """ sets the indentation baseline — all common leading whitespace is stripped.

Java 14

1. Switch Expressions *(Final — JEP 361)*

Stable and production-ready. The yield keyword returns a value from a block case:

int result = switch (input) {
    case "one"   -> 1;
    case "two"   -> 2;
    default -> {
        System.out.println("unknown: " + input);
        yield -1;   // yield in block cases
    }
};
System.out.println(result); // -1 (if input is unknown)

2. Helpful NullPointerExceptions *(JEP 358)*

The JVM now tells you exactly which variable was null, not just the line.

Before Java 14:

Exception in thread "main" java.lang.NullPointerException
    at Example.main(Example.java:5)

Java 14+:

Exception in thread "main" java.lang.NullPointerException:
    Cannot invoke "String.length()" because "str" is null
    at Example.main(Example.java:5)
String str = null;
int len = str.length(); // now says: "str" is null — no guessing!

For chained calls, it's even more specific:

user.getAddress().getCity().toUpperCase();
// "Cannot invoke String.toUpperCase() because getCity() returned null"

Java 15

1. Text Blocks *(Final — JEP 378)*

Text blocks are now stable. Common in tests, SQL, JSON, and config:

String sql = """
        SELECT u.name, u.email
        FROM users u
        WHERE u.active = true
        ORDER BY u.name
        """;
String json = """
        {
          "status": "ok",
          "count": 3
        }
        """;

String methods added to work with text blocks:

// formatted() — like String.format but as an instance method
String msg = """
        Hello, %s! You have %d messages.
        """.formatted("Alice", 5);
System.out.println(msg);
// Hello, Alice! You have 5 messages.

2. Hidden Classes *(JEP 371)*

Classes that cannot be used directly by other classes — intended for frameworks that generate classes at runtime (proxies, lambdas, etc.). Not typically used in application code directly, but important for framework authors.

// Frameworks use this instead of Unsafe.defineAnonymousClass()
MethodHandles.Lookup lookup = MethodHandles.lookup();
byte[] classBytes = ...; // generated bytecode
Class<?> hidden = lookup.defineHiddenClass(classBytes, true).lookupClass();
// hidden class is not discoverable by name — garbage collected when no longer needed

Java 16

1. Records

A record is a compact, immutable data class. The compiler auto-generates constructor, getters (name() not getName()), equals(), hashCode(), and toString().

Before Java 16:

public class Point {
    private final int x;
    private final int y;
    public Point(int x, int y) { this.x = x; this.y = y; }
    public int getX() { return x; }
    public int getY() { return y; }
    @Override public boolean equals(Object o) { ... }
    @Override public int hashCode() { ... }
    @Override public String toString() { return "Point[x=" + x + ", y=" + y + "]"; }
}

Java 16+:

public record Point(int x, int y) {}
// Usage
Point p = new Point(3, 5);
System.out.println(p);        // Point[x=3, y=5]
System.out.println(p.x());    // 3
System.out.println(p.y());    // 5
Point p2 = new Point(3, 5);
System.out.println(p.equals(p2)); // true

Records can have compact constructors for validation:

public record Range(int min, int max) {
    Range {  // compact constructor — no parameter list needed
        if (min > max) throw new IllegalArgumentException("min > max");
    }
}

2. Pattern Matching for `instanceof`

Eliminates the manual cast after an instanceof check.

Before Java 16:

Object obj = "Hello";
if (obj instanceof String) {
    String s = (String) obj;   // manual cast
    System.out.println(s.toUpperCase());
}

Java 16+:

Object obj = "Hello";
if (obj instanceof String s) {
    System.out.println(s.toUpperCase()); // s is already a String
}
// Output: HELLO
// Works in expressions too:
Object val = 42;
String result = (val instanceof Integer i) ? "int: " + i : "other";
System.out.println(result); // int: 42

3. `Stream.toList()`

A shorter, unmodifiable alternative to collect(Collectors.toList()).

Before Java 16:

import java.util.stream.Collectors;
List<String> names = stream.collect(Collectors.toList());

Java 16+:

List<String> names = List.of("Alice", "Bob", "Charlie");
List<String> filtered = names.stream()
    .filter(n -> n.startsWith("A"))
    .toList();
System.out.println(filtered); // [Alice]
// Note: result is unmodifiable (unlike Collectors.toList())

Java 17

*(Long-Term Support release — the most common production baseline)*

1. Sealed Classes and Interfaces

Restricts which classes may extend or implement a type. Works perfectly with pattern matching — the compiler knows the full set of subtypes.

Before Java 17:

// Anyone can extend — no enforcement
public abstract class Shape {}
public class Circle extends Shape {}
public class Rectangle extends Shape {}
public class Triangle extends Shape {}  // can't prevent this

Java 17+:

// Shape.java
public sealed interface Shape permits Circle, Rectangle, Triangle {}
public record Circle(double radius) implements Shape {}
public record Rectangle(double w, double h) implements Shape {}
public record Triangle(double base, double height) implements Shape {}
// Usage — compiler enforces exhaustiveness in switch
double area = switch (shape) {
    case Circle c    -> Math.PI * c.radius() * c.radius();
    case Rectangle r -> r.w() * r.h();
    case Triangle t  -> 0.5 * t.base() * t.height();
};
System.out.println(area); // depends on shape

2. Text Blocks

*(Finalized in Java 15, but Java 17 is the LTS baseline where most teams first adopted it.)*

Multi-line string literals without escape sequences. Leading whitespace is automatically stripped based on the closing """ position.

Before:

String json = "{\n" +
    "  \"name\": \"Alice\",\n" +
    "  \"age\": 30\n" +
    "}";

Java 15+ (standard in 17 LTS):

String json = """
        {
          "name": "Alice",
          "age": 30
        }
        """;
System.out.println(json);
// Output:
// {
//   "name": "Alice",
//   "age": 30
// }

Useful for SQL, HTML, JSON, and YAML in tests and configs.

3. Strong Encapsulation of JDK Internals

What changed: Access to internal JDK APIs (e.g., sun.misc.Unsafe via reflection) is now blocked by default. Code relying on --add-opens flags or internal packages will break.

Action: Replace uses of sun.* / com.sun.* with official APIs. Libraries like Apache Commons and Spring were updated — upgrade your dependencies.

Java 18

1. UTF-8 by Default

Before Java 18, the default charset depended on the OS locale (e.g., Windows-1252 on some Windows systems). This caused subtle bugs when moving apps between environments.

Before Java 18:

// Could print different things depending on OS locale
System.out.println(Charset.defaultCharset()); // e.g., windows-1252

Java 18+:

System.out.println(Charset.defaultCharset()); // UTF-8 (always)

If your app explicitly uses StandardCharsets.UTF_8 everywhere, this change has no impact. If it relied on the implicit default, files and streams may now read/write differently.

2. Simple Web Server

A minimal HTTP file server for development, testing, and prototyping — no frameworks needed.

CLI usage:

# Serve current directory on port 8080
jwebserver
# Custom port and directory
jwebserver -p 9000 -d /tmp/my-site

Programmatic usage:

import com.sun.net.httpserver.*;
import java.net.InetSocketAddress;
import java.nio.file.Path;
var server = SimpleFileServer.createFileServer(
    new InetSocketAddress(8080),
    Path.of("/tmp/my-site"),
    SimpleFileServer.OutputLevel.VERBOSE
);
server.start();
// Now browse http://localhost:8080

Not intended for production — use for quick local testing, CI artifact serving, or mock backends.

Java 19

1. Virtual Threads *(Preview — JEP 425)*

Part of Project Loom. Virtual threads are lightweight threads managed by the JVM, not the OS. You can create millions of them cheaply — ideal for I/O-bound workloads.

Before Java 19 — platform threads (expensive):

// Each thread maps to an OS thread — limited to thousands
ExecutorService pool = Executors.newFixedThreadPool(200);
for (int i = 0; i < 10_000; i++) {
    pool.submit(() -> {
        Thread.sleep(Duration.ofSeconds(1)); // blocks OS thread
        System.out.println("done");
    });
}

Java 19+ — virtual threads (cheap):

// Each task gets its own virtual thread — JVM manages scheduling
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 10_000; i++) {
        executor.submit(() -> {
            Thread.sleep(Duration.ofSeconds(1)); // does NOT block OS thread
            System.out.println("done");
        });
    }
}
// Or directly:
Thread vt = Thread.ofVirtual().start(() -> System.out.println("virtual!"));
vt.join();
// Output: virtual!

Virtual threads shine for request-per-thread server models (e.g., Spring Boot, Tomcat). They do NOT speed up CPU-bound code.

2. Structured Concurrency *(Incubator — JEP 428)*

Treats multiple concurrent tasks as a single unit of work. If one task fails, others are cancelled. Eliminates common pitfalls of unstructured Future management.

import jdk.incubator.concurrent.StructuredTaskScope;
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    Future<String> user  = scope.fork(() -> fetchUser(id));
    Future<String> order = scope.fork(() -> fetchOrder(id));
    scope.join();           // wait for both
    scope.throwIfFailed();  // propagate any exception
    System.out.println(user.get() + " / " + order.get());
}
// If fetchUser() throws, fetchOrder() is cancelled automatically

Java 20

1. Record Patterns *(Preview 2 — JEP 432)*

Allows destructuring a record directly inside a pattern match, binding its components to variables.

Before (Java 16 records + instanceof):

if (obj instanceof Point p) {
    int x = p.x();  // manual accessor calls
    int y = p.y();
    System.out.println(x + ", " + y);
}

Java 20+ — record pattern:

if (obj instanceof Point(int x, int y)) {
    System.out.println(x + ", " + y); // x and y bound directly
}
// Output: 3, 5  (if obj = new Point(3, 5))

Nested records also work:

record Segment(Point start, Point end) {}
if (shape instanceof Segment(Point(int x1, int y1), Point(int x2, int y2))) {
    System.out.println("from " + x1 + "," + y1 + " to " + x2 + "," + y2);
}

2. Scoped Values *(Incubator — JEP 429)*

A safer, more efficient alternative to ThreadLocal, designed for virtual threads. A scoped value is set once, immutably, and is visible only within a bounded scope.

ThreadLocal problem: mutable, inheritable, leaks memory in thread pools.

Scoped Values:

import jdk.incubator.concurrent.ScopedValue;
static final ScopedValue<String> CURRENT_USER = ScopedValue.newInstance();
// Set in outer scope — readable by all code called within the run block
ScopedValue.where(CURRENT_USER, "alice").run(() -> {
    System.out.println(CURRENT_USER.get()); // alice
    processRequest();  // can also read CURRENT_USER.get() here
});
// CURRENT_USER.get() outside the run() block → NoSuchElementException

Java 21

*(Long-Term Support — current recommended LTS)*

1. Virtual Threads *(Final — JEP 444)*

Virtual threads are now production-ready with no feature flags. Enable in Spring Boot with one property:

spring.threads.virtual.enabled=true

Or in plain Java:

try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    IntStream.range(0, 100_000).forEach(i ->
        executor.submit(() -> {
            Thread.sleep(Duration.ofMillis(100));
            System.out.println(Thread.currentThread().isVirtual()); // true
        })
    );
}

2. Sequenced Collections *(JEP 431)*

New interfaces (SequencedCollection, SequencedSet, SequencedMap) add first/last access and reversed() to List, LinkedHashSet, LinkedHashMap, etc.

Before Java 21:

List<String> list = List.of("a", "b", "c");
String first = list.get(0);              // fragile index
String last  = list.get(list.size()-1);  // verbose
LinkedHashMap<String,Integer> map = new LinkedHashMap<>();
// No standard way to get first/last entry

Java 21+:

List<String> list = List.of("a", "b", "c");
System.out.println(list.getFirst()); // a
System.out.println(list.getLast());  // c
List<String> rev = list.reversed().stream().toList();
System.out.println(rev); // [c, b, a]
LinkedHashMap<String,Integer> map = new LinkedHashMap<>();
map.put("x", 1); map.put("y", 2);
System.out.println(map.firstEntry()); // x=1
System.out.println(map.lastEntry());  // y=2

3. Pattern Matching for `switch` *(Final — JEP 441)*

Type patterns in switch, with exhaustiveness checking for sealed types.

Before Java 21:

static String format(Object obj) {
    if (obj instanceof Integer i) return "int: " + i;
    if (obj instanceof Double d)  return "double: " + d;
    if (obj instanceof String s)  return "string: " + s;
    return "unknown";
}

Java 21+:

static String format(Object obj) {
    return switch (obj) {
        case Integer i -> "int: " + i;
        case Double d  -> "double: " + d;
        case String s  -> "string: " + s;
        case null      -> "null";
        default        -> "unknown";
    };
}
System.out.println(format(42));      // int: 42
System.out.println(format(3.14));    // double: 3.14
System.out.println(format("hi"));    // string: hi

Guarded patterns — add conditions with when:

return switch (obj) {
    case Integer i when i < 0 -> "negative int: " + i;
    case Integer i             -> "positive int: " + i;
    default                    -> "other";
};

4. Record Patterns *(Final — JEP 440)*

Now stable — destructure records inline in switch and instanceof:

record Point(int x, int y) {}
Object obj = new Point(0, 5);
String desc = switch (obj) {
    case Point(int x, int y) when x == 0 -> "on Y-axis at y=" + y;
    case Point(int x, int y)             -> "at (" + x + "," + y + ")";
    default                              -> "not a point";
};
System.out.println(desc); // on Y-axis at y=5

Java 22

1. Unnamed Variables and Patterns *(Final — JEP 456)*

Use _ to mark variables or pattern components you intentionally don't use. Clarifies intent and avoids "unused variable" warnings.

Unnamed catch variable:

// Before
try { riskyOp(); } catch (IOException e) { log("failed"); } // e unused
// Java 22+
try { riskyOp(); } catch (IOException _) { log("failed"); }

Unnamed loop variable:

int count = 0;
for (var _ : list) count++;  // don't care about the element
System.out.println("count: " + count);

Unnamed pattern component:

if (obj instanceof Point(int x, _)) {
    System.out.println("x = " + x); // only care about x
}

2. Statements Before `super()` *(Preview — JEP 447)*

Allows validation or computation before calling super() in a constructor. Previously any code before super() was a compile error.

Before Java 22:

class PositivePoint extends Point {
    PositivePoint(int x, int y) {
        super(x, y); // must be first — can't validate before this
        if (x < 0 || y < 0) throw new IllegalArgumentException(); // too late?
    }
}

Java 22+ (preview):

class PositivePoint extends Point {
    PositivePoint(int x, int y) {
        if (x < 0 || y < 0) throw new IllegalArgumentException("must be positive");
        super(x, y);  // called after validation
    }
}

3. Foreign Function & Memory API *(Final — JEP 454)*

Safe, efficient access to native code and off-heap memory — replaces unsafe uses of JNI and sun.misc.Unsafe.

import java.lang.foreign.*;
import java.lang.invoke.MethodHandle;
// Call C's strlen() from Java — no JNI boilerplate
try (Arena arena = Arena.ofConfined()) {
    Linker linker = Linker.nativeLinker();
    SymbolLookup stdlib = linker.defaultLookup();
    MethodHandle strlen = linker.downcallHandle(
        stdlib.find("strlen").get(),
        FunctionDescriptor.of(ValueLayout.JAVA_LONG, ValueLayout.ADDRESS)
    );
    MemorySegment str = arena.allocateFrom("Hello");
    long len = (long) strlen.invoke(str);
    System.out.println(len); // 5
}

Java 23

1. Primitive Types in Patterns *(Preview — JEP 455)*

Extends pattern matching to work with primitive types in switch and instanceof, unifying the type system further.

Before Java 23:

// Primitives couldn't be used as switch patterns — required boxing
Object obj = 42;
if (obj instanceof Integer i && i > 0) { ... }

Java 23+ (preview):

// Primitives directly in switch (no boxing required)
switch (value) {  // value is int or long
    case int i when i < 0    -> System.out.println("negative");
    case int i when i == 0   -> System.out.println("zero");
    case int i               -> System.out.println("positive: " + i);
}
// Also works in instanceof
int n = 42;
if (n instanceof int i && i > 10) {
    System.out.println("big int: " + i); // big int: 42
}

2. Markdown Documentation Comments *(JEP 467)*

Write Javadoc using Markdown (/// comments) instead of HTML tags. More readable in source, same rendered output.

Before Java 23:

/**
 * Returns the {@code name} of this user.
 * <p>
 * The name is <b>never</b> {@code null}.
 * </p>
 * @return the user's name
 */
public String getName() { ... }

Java 23+:

/// Returns the `name` of this user.
///
/// The name is **never** `null`.
///
/// @return the user's name
public String getName() { ... }

3. ZGC: Generational Mode Default

The Z Garbage Collector now defaults to generational mode, improving throughput and reducing pause times for most applications with no code changes required.

# Before Java 23 — had to opt in
java -XX:+UseZGC -XX:+ZGenerational -jar app.jar
# Java 23+ — generational is default when ZGC is selected
java -XX:+UseZGC -jar app.jar

Java 24

1. Stream Gatherers *(Final — JEP 485)*

Custom intermediate Stream operations via Stream.gather(Gatherer). The built-in Gatherers class provides common ones; you can also write your own.

Before Java 24 — no clean way to do sliding windows:

// Had to collect to list first, then manually slice
List<Integer> nums = List.of(1, 2, 3, 4, 5);
List<List<Integer>> windows = new ArrayList<>();
for (int i = 0; i <= nums.size() - 3; i++) {
    windows.add(nums.subList(i, i + 3));
}
System.out.println(windows); // [[1, 2, 3], [2, 3, 4], [3, 4, 5]]

Java 24+:

import java.util.stream.Gatherers;
List<Integer> nums = List.of(1, 2, 3, 4, 5);
// Fixed-size windows (non-overlapping)
List<List<Integer>> fixed = nums.stream()
    .gather(Gatherers.windowFixed(2))
    .toList();
System.out.println(fixed); // [[1, 2], [3, 4], [5]]
// Sliding windows
List<List<Integer>> sliding = nums.stream()
    .gather(Gatherers.windowSliding(3))
    .toList();
System.out.println(sliding); // [[1, 2, 3], [2, 3, 4], [3, 4, 5]]
// Scan (running sum)
List<Integer> running = nums.stream()
    .gather(Gatherers.scan(() -> 0, Integer::sum))
    .toList();
System.out.println(running); // [1, 3, 6, 10, 15]

Custom gatherer example — take until condition:

Gatherer<Integer, ?, Integer> takeUntilOver10 = Gatherer.ofSequential(
    () -> new boolean[]{false},
    (state, elem, downstream) -> {
        if (state[0]) return false;
        if (elem > 10) { state[0] = true; return false; }
        return downstream.push(elem);
    }
);
List<Integer> result = Stream.of(2, 5, 8, 12, 3)
    .gather(takeUntilOver10)
    .toList();
System.out.println(result); // [2, 5, 8]

2. Class-File API *(Final — JEP 484)*

A standard API for reading, writing, and transforming Java .class files. Replaces third-party libraries like ASM and Javassist for many use cases.

import java.lang.classfile.*;
import java.lang.constant.*;
// Build a class programmatically
byte[] bytes = ClassFile.of().build(
    ClassDesc.of("com.example.Hello"),
    classBuilder -> classBuilder
        .withMethod("greet", MethodTypeDesc.of(CD_void), ACC_PUBLIC,
            methodBuilder -> methodBuilder.withCode(code -> code
                .getstatic(CD_System, "out", CD_PrintStream)
                .ldc("Hello from generated class!")
                .invokevirtual(CD_PrintStream, "println",
                    MethodTypeDesc.of(CD_void, CD_String))
                .return_()
            ))
);
// bytes can be loaded via ClassLoader.defineClass(...)

3. Ahead-of-Time Class Loading & Linking *(JEP 483)*

Caches the result of class loading and linking across JVM runs, reducing startup time without code changes.

# Step 1: training run — JVM records what it loads
java -XX:AOTMode=record -XX:AOTConfiguration=app.aotconf -jar app.jar
# Step 2: use cached data on subsequent runs
java -XX:AOTMode=on -XX:AOTConfiguration=app.aotconf -jar app.jar
# Noticeably faster startup on second run

Java 25

*(Long-Term Support — September 2025)*

1. Module Import Declarations *(Final — JEP 494)*

Import all packages exported by a module in one line — no need to list each package separately.

Before Java 25:

import java.util.List;
import java.util.Map;
import java.util.stream.Collectors;
import java.util.function.Function;
// ... many more java.base imports

Java 25+:

import module java.base;   // imports all exported packages from java.base
// Now use List, Map, Stream, etc. without individual imports
List<String> names = List.of("Alice", "Bob");
Map<String, Integer> lengths = names.stream()
    .collect(Collectors.toMap(Function.identity(), String::length));
System.out.println(lengths); // {Alice=5, Bob=3}

2. Stable Values *(Preview — JEP 502)*

Lazily initialized, effectively-final values with safe publication. Better than volatile + double-checked locking, simpler than Supplier caching.

Before Java 25 — double-checked locking boilerplate:

private volatile Config config;
public Config getConfig() {
    if (config == null) {
        synchronized (this) {
            if (config == null) config = loadConfig();
        }
    }
    return config;
}

Java 25+ (preview):

import java.lang.StableValue;
private final StableValue<Config> config = StableValue.of();
public Config getConfig() {
    return config.orElseSet(this::loadConfig); // thread-safe, initialized once
}

Also useful for lazy fields in records and interfaces.

3. Primitive Classes / Value Objects *(Preview — JEP 401)*

Part of Project Valhalla. value classes have no identity — two instances with the same fields are indistinguishable. The JVM can flatten them into arrays and avoid heap allocation.

Before Java 25 — all objects have identity:

record Point(int x, int y) {} // still a reference type with identity
Point a = new Point(1, 2);
Point b = new Point(1, 2);
System.out.println(a == b);       // false — different references
System.out.println(a.equals(b));  // true — same values

Java 25+ (preview):

value record Point(int x, int y) {}  // no identity — only values matter
Point a = new Point(1, 2);
Point b = new Point(1, 2);
// == is now equivalent to equals() — same field values, same identity
System.out.println(a == b); // true
// JVM can store Point[] without boxing — huge performance win for data-heavy code
Point[] arr = new Point[1_000_000]; // flattened in memory, no object headers
